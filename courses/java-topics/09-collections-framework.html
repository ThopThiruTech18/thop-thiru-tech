Java Collections Framework (Zero → Hero)

What is a Collection?

A Collection is an object that stores multiple elements. Collections are dynamic (size can grow/shrink) and are part of java.util.

Why Collections?

  • Dynamic size
  • Ready-made classes and implementations
  • Powerful utility methods
  • Easy data manipulation

Array vs Collection

FeatureArrayCollection

1. List (Ordered, duplicates allowed)

ArrayList

List list = new ArrayList<>();
list.add("Java");
list.add("Spring");
list.add("Java");
System.out.println(list);

Fast random access, slower insert/delete in middle.

LinkedList

List list = new LinkedList<>();
list.add("Java");
list.add("Spring");

Fast insertion/deletion, slower random access.

Vector / Stack (Legacy)

Vector is synchronized (thread-safe). Stack provides LIFO operations.

2. Set (No duplicates)

HashSet

Set set = new HashSet<>();
set.add(10);
set.add(20);
set.add(10);
System.out.println(set);

Fast, but no guaranteed order.

LinkedHashSet & TreeSet

LinkedHashSet preserves insertion order. TreeSet keeps elements sorted.

3. Queue (FIFO)

PriorityQueue

Queue q = new PriorityQueue<>();
q.add(30); q.add(10); q.add(20);
System.out.println(q.poll()); // 10

Deque / ArrayDeque

Deque dq = new ArrayDeque<>();
dq.addFirst(10); dq.addLast(20);

Map (Key → Value)

Map map = new HashMap<>();
map.put(1, "Java");
map.put(2, "Spring");

HashMap is fast (no order). LinkedHashMap preserves insertion order. TreeMap sorts by key.

Common Methods

list.add();
list.remove();
list.contains();
list.size();
list.isEmpty();
list.clear();

Iterating Collections

for-each

for (String s : list) {
    System.out.println(s);
}

Iterator

Iterator it = list.iterator();
while (it.hasNext()) System.out.println(it.next());

Advanced

Comparable

class Student implements Comparable {
    public int compareTo(Student s) { return this.id - s.id; }
}

Comparator

Comparator c = (s1, s2) -> s1.name.compareTo(s2.name);

Collections Utility

Collections.sort(list);
Collections.reverse(list);
Collections.max(list);

Real-time Usage

  • Store DB results
  • Cache data
  • User sessions
  • API responses

Interview One-Liners

  • List → Order + duplicates
  • Set → Unique elements
  • Map → Key–value
  • HashMap → Fastest
  • TreeMap → Sorted

Final Summary

  • Collections are dynamic
  • Provide ready-made data structures
  • Backbone of Java applications
