What is an Exception?

An exception is an unexpected problem that occurs during program execution and disturbs the normal flow of the program.

Real-life example

ATM card inserted but no balance → program should handle it instead of crashing.

Types of Exceptions

  1. Checked Exceptions (compile-time)
  2. Unchecked Exceptions (runtime)
  3. Errors (JVM problems)

Checked Exceptions

Checked at compile time — must be handled or declared (e.g., IOException, SQLException, FileNotFoundException).

FileReader fr = new FileReader("data.txt"); // may require throws or try-catch

Unchecked Exceptions

Occur at runtime and are not checked by the compiler (e.g., NullPointerException, ArithmeticException, ArrayIndexOutOfBoundsException).

int a = 10 / 0; // ArithmeticException at runtime

Errors

Serious JVM problems that are usually not recoverable (e.g., StackOverflowError, OutOfMemoryError).

Exception Hierarchy

Handling Exceptions

try–catch

Wrap risky code in try, handle in catch:

try {
		int a = 10 / 0;
} catch (ArithmeticException e) {
		System.out.println("Cannot divide by zero");
}

Multiple catch blocks

Place child exceptions first, parent last:

try {
		int[] a = {1, 2};
		System.out.println(a[5]);
} catch (ArrayIndexOutOfBoundsException e) {
		System.out.println("Index error");
} catch (Exception e) {
		System.out.println("General exception");
}

finally

Executes always (used for cleanup):

try {
		// code
} catch (Exception e) {
		// handle
} finally {
		// always executed
}

Try-with-resources (Java 7+)

Automatically closes resources that implement AutoCloseable:

try (FileReader fr = new FileReader("data.txt")) {
		System.out.println("Reading file");
} catch (Exception e) {
		e.printStackTrace();
}

throw vs throws

throw — create/throw an exception manually.

throw new ArithmeticException("Invalid operation");

throws — declare that a method may throw exceptions.

void readFile() throws IOException {
		FileReader fr = new FileReader("data.txt");
}

Custom Exceptions

Create domain-specific exceptions by extending Exception or RuntimeException:

class InvalidAgeException extends Exception {
		InvalidAgeException(String msg) { super(msg); }
}

// usage
if (age < 18) {
		throw new InvalidAgeException("Not eligible to vote");
}

Best Practices

  • Catch specific exceptions
  • Never swallow exceptions (don't use empty catch)
  • Use finally or try-with-resources for cleanup
  • Don't use exceptions for flow control

Interview One-Line Answers

  • Exception → Runtime abnormal condition
  • Checked → Compile-time
  • Unchecked → Runtime
  • throw → Create/throw exception
  • throws → Declare exception

Final Summary

  • Exceptions prevent application crash when handled
  • Proper handling improves reliability and maintainability
  • Core topic for interviews and real-world projects
